## 一句话项目名称

ClinicFlow AI：面向中小诊所与个人工作室的多模态流程助手

## 一句话项目介绍

我正在开发一个基于大模型的诊所业务流程助手，帮助用户把聊天记录、表单、图片材料和语音备注整理为结构化任务、标准化文档和可执行的后续动作，降低人工整理成本，提高交付效率。

## 项目详细描述

我正在做一个真实可落地的 AI 应用原型，名称是 ClinicFlow AI。它面向诊所、咨询工作室、个人服务团队这类文档和沟通任务很多、但缺少技术团队的小型组织。

项目的核心流程是：

1. 用户上传或粘贴多种输入，包括聊天记录、需求说明、图片资料、表单内容、语音转写文本。
2. 系统自动识别用户意图和关键信息，抽取结构化字段，例如客户信息、时间节点、任务优先级、交付要求、风险项。
3. 系统自动生成标准输出，包括任务清单、沟通摘要、项目说明、阶段计划、提醒事项和可继续编辑的文档草稿。
4. 系统支持多轮迭代，用户可以继续追问、补充材料、修改目标，模型会持续更新输出结果。

这个项目不是单次 demo，而是希望逐步做成一个可长期使用的生产力工具。当前我会先完成以下几个版本：

- V1：文本整理与摘要生成
- V2：图文混合材料理解与字段提取
- V3：多轮任务编排与模板化输出
- V4：接入 Claude Code / Cursor 等开发工具，自动生成部分前后端原型和工作流脚本

我申请 MiMo Token 计划，是因为项目需要持续、高频地测试长上下文、多轮推理、结构化输出稳定性，以及图文混合理解能力。仅靠少量免费额度很难完成完整评估和真实使用验证。

## 目标用户

- 中小诊所

- 个人工作室
- 需要频繁整理资料、写说明、做流程交付的独立从业者

## 为什么需要 MiMo

我希望重点测试 MiMo 在以下场景中的能力：

- 长上下文整理：一次处理多段聊天记录、需求说明和补充材料
- 结构化抽取：从混乱输入中提取稳定字段
- 多轮任务规划：把需求变成可执行步骤
- 多模态理解：后续会加入图片材料、截图、表单截图等输入
- 成本可控的高频调用：需要进行大量 prompt 调优、模板调优和回归测试

MiMo 的 Token Plan 对我有明显帮助，因为它可以让我持续进行真实场景测试，而不是只做少量展示性质的样例。

## 预计使用方式

- 开发阶段：通过 API 在本地原型中调用 MiMo，调试任务拆解、摘要、抽取、重写与问答能力
- 工具接入：在 Claude Code、Cursor 或自建脚本工作流中接入 MiMo，用于生成模板、校验输出、执行批量内容处理
- 评估阶段：构建一组固定测试集，对不同 prompt、不同流程版本进行反复评测

## 预计 Token 消耗说明

项目会持续消耗较多 Token，原因如下：

- 单次任务通常包含较长上下文
- 需要多轮改写和追问
- 需要对同一份材料做多版本比较
- 需要反复测试不同模板和工作流
- 后续会增加图片理解和多模态输入

因此我希望获得较高档位的支持，以便完成稳定性测试、原型开发和小范围真实试用。
